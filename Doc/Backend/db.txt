Description
The db.py module serves as the database management component of the Crime Rate Prediction System. It is responsible for establishing database connections, creating required tables, storing user information, managing prediction records, and retrieving data when needed. This module acts as a bridge between the application interface and the database, ensuring secure and efficient data storage and retrieval.

Features
Database Connection
Establishes connection with MySQL database.

Handles database communication.

Maintains data integrity.

User Management
Stores user registration details.

Supports user authentication.

Manages user login records.

Prediction Storage
Saves crime prediction results.

Maintains prediction history.

Stores state-wise forecast records.

Data Retrieval
Retrieves stored prediction records.

Fetches user information.

Supports report generation.

Table Initialization
Automatically creates required database tables.

Ensures database structure consistency.

Common Errors
Error 1: Database Connection Failed
The error message is Can't connect to MySQL server. This happens because the MySQL server is not running or database credentials are incorrect. To solve this, start the MySQL service, verify the username and password, and check the database host and port settings.

Error 2: Access Denied
The error message is Access denied for user. This occurs due to an incorrect database username or password. To solve this, update the credentials in db.py with valid database login details.

Error 3: Database Not Found
The error message is Unknown database. This happens because the specified database does not exist. To solve this, create the required database manually using the SQL command CREATE DATABASE crime_prediction;.

Error 4: Table Not Found
The error message is Table doesn't exist. This occurs because the required tables were not created. To solve this, run the init_db() function to create tables automatically.

Error 5: Insert Query Failed
The error message is Error while inserting data. This is caused by an invalid data format or database constraints. To solve this, verify input values, check the table schema, and validate the SQL query.

Debugging Guide
To debug your system, you can use print statements to verify different operations:

Use print(Database Connected Successfully) to check the connection.

Use print(Tables Created Successfully) to check table creation.

Use print(User Registered Successfully) for user registration.

Use print(Login Successful) for user login.

Use print(Prediction Saved Successfully) to check if predictions are saved.

Use print(Records Retrieved Successfully) to check if data retrieval works.

Database Workflow
The application starts by connecting to the database. Next, it initializes the tables. Then, it proceeds to store user data and prediction results. Finally, it retrieves records to display the data.

Database Tables
Users Table
Stores registered user information. Fields include id, username, email, and password.

Predictions Table
Stores generated prediction records. Fields include prediction_id, user_id, state, year, predicted_crime, and prediction_date.
