# Database Module (db.py)

## Description

The `db.py` module serves as the database management component of the Crime Rate Prediction System. It is responsible for establishing database connections, creating required tables, storing user information, managing prediction records, and retrieving data when needed.

This module acts as a bridge between the application interface and the database, ensuring secure and efficient data storage and retrieval.

---

# Features

## Database Connection

- Establishes connection with MySQL database.
- Handles database communication.
- Maintains data integrity.

## User Management

- Stores user registration details.
- Supports user authentication.
- Manages user login records.

## Prediction Storage

- Saves crime prediction results.
- Maintains prediction history.
- Stores state-wise forecast records.

## Data Retrieval

- Retrieves stored prediction records.
- Fetches user information.
- Supports report generation.

## Table Initialization

- Automatically creates required database tables.
- Ensures database structure consistency.

# Common Errors

## Error 1: Database Connection Failed

### Error Message

```text
Can't connect to MySQL server
```

### Reason

The MySQL server is not running or database credentials are incorrect.

### Solution

- Start MySQL service.
- Verify username and password.
- Check database host and port settings.

---

## Error 2: Access Denied

### Error Message

```text
Access denied for user
```

### Reason

Incorrect database username or password.

### Solution

Update credentials in:

```python
db.py
```

with valid database login details.

---

## Error 3: Database Not Found

### Error Message

```text
Unknown database
```

### Reason

The specified database does not exist.

### Solution

Create the required database manually:

```sql
CREATE DATABASE crime_prediction;
```

---

## Error 4: Table Not Found

### Error Message

```text
Table doesn't exist
```

### Reason

Required tables were not created.

### Solution

Run:

```python
init_db()
```

to create tables automatically.

---

## Error 5: Insert Query Failed

### Error Message

```text
Error while inserting data
```

### Reason

Invalid data format or database constraints.

### Solution

- Verify input values.
- Check table schema.
- Validate SQL query.
