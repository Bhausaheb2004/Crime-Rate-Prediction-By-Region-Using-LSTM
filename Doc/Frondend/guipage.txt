GUI Module (guipage.py)
Description
The guipage.py file serves as the main dashboard of the Crime Rate Prediction System. It provides a graphical user interface that allows users to interact with the application, select states, generate crime predictions, view historical crime trends, and visualize prediction results through graphs.

Features
Crime Prediction Dashboard
Interactive GUI built using Tkinter.

User-friendly interface.

State selection using a dropdown menu.

Crime trend visualization.

State Selection
Displays all available states from the dataset.

Allows users to choose a state for analysis.

Crime Prediction
Generates future crime predictions.

Displays historical and predicted crime data.

Stores prediction results in the database.

Graph Visualization
Shows historical crime trends.

Displays future prediction trends.

Uses Matplotlib for chart generation.

Navigation
Opens Crime Wise Prediction module.

Provides Exit functionality.

Common Errors
Error 1: State Not Selected
The error message is Select a state. This happens when the user clicks Predict without selecting a state. Solve this by selecting a state from the dropdown list before clicking Predict.

Error 2: Dataset File Not Found
The error message is FileNotFoundError. This occurs because the crime dataset path is incorrect. Verify the DATA_PATH variable and ensure the CSV file exists in the correct directory.

Error 3: State Wise Module Not Opening
The error message is AttributeError: 'CrimePredictionApp' object has no attribute 'open_state_wise'. This happens when a button calls a function that does not exist. Solve this by defining the method inside the class and ensuring the command attribute is linked correctly.

Error 4: Database Connection Error
This occurs if the system is unable to connect to the MySQL database. Verify your database credentials, ensure the MySQL service is running, and check your table creation scripts.

Error 5: Graph Not Displaying
This happens if the Matplotlib data is missing or invalid. Verify dataset contents, check the graph generation code, and ensure that Matplotlib is installed properly.

Debugging Guide
To debug guipage.py, use the following checks:

Use print(Dataset Loaded Successfully) to verify data loading.

Use print(Selected State: state) to verify the current selection.

Use print(Prediction Started) and print(Prediction Completed) to track the prediction process.

Use print(State Wise Prediction Button Clicked) to check button functionality.

Use print(Prediction Saved Successfully) to verify database operations.

GUI Workflow
The user opens the dashboard and selects a state. Upon clicking Predict, the system loads historical data, generates a future prediction, displays the results, and shows the corresponding graph.

File Location
The file is located in the Frontend folder as guipage.py. The GUI module acts as the central interface connecting dataset analysis, prediction generation, graph visualization, and navigation to other project modules.
