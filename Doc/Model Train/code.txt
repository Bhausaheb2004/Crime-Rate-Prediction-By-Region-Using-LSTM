Model Training Module (run_training.py / code.py)
Description
The Model Training Module is the core component of the Crime Rate Prediction System. It is responsible for training Machine Learning and Deep Learning models using historical crime data collected from various states and union territories of India. This module processes the dataset, performs data preprocessing, trains the prediction model, evaluates performance, and saves the trained model for future predictions. The primary goal is to generate an accurate forecasting model capable of predicting future crime rates based on historical crime trends.

Features
Dataset Loading
Loads the crime dataset from a CSV file.

Reads historical crime records and validates input data.

Data Preprocessing
Handles missing values and removes duplicate records.

Normalizes data and converts it into a time-series format.

Model Training
Trains the LSTM (Long Short-Term Memory) model.

Learns historical crime patterns and generates the forecasting model.

Model Evaluation
Calculates prediction accuracy, generates loss graphs, and measures model performance.

Model Saving
Saves trained models and their weights to enable future prediction without retraining.

Machine Learning Model: LSTM
This project utilizes LSTM, a specialized Recurrent Neural Network (RNN) designed for time-series forecasting.

Advantages
Captures long-term dependencies.

Handles sequential data efficiently.

Suitable for crime forecasting.

Produces accurate predictions by learning complex patterns from historical data.

Training Workflow
The training process follows this sequence:

Load the crime dataset.

Clean the data.

Perform data preprocessing.

Conduct feature engineering.

Split the data into training and testing sets.

Train the LSTM model.

Evaluate the model performance.

Analyze accuracy and loss.

Save the trained model for production use.

Common Errors
Error 1: Dataset Not Found
The error message is FileNotFoundError. This occurs when the dataset path is incorrect. Verify that the crime_data_india.csv file exists in the specified directory.

Error 2: TensorFlow Not Installed
The error message is ModuleNotFoundError: No module named 'tensorflow'. This means the TensorFlow package is missing. Install it using the command: pip install tensorflow.

Error 3: Insufficient Training Data
The error message is ValueError. This happens if the dataset contains too few records for effective training. Use a larger dataset with more historical data.

Error 4: Model Training Failed
This is caused by incorrect data preprocessing or invalid input shapes. Verify your dataset columns, check your preprocessing steps, and validate the input dimensions.

Error 5: Model Save Failed
The error message is PermissionError. The application likely cannot write to the model directory. Verify folder permissions and the specified save path.

Debugging Guide
To debug the module, use these print statements:

Use print(Dataset Loaded Successfully) to verify loading.

Use print(df.shape) to check dataset dimensions.

Use print(df.isnull().sum()) to identify missing values.

Use print(X_train.shape) and print(y_train.shape) to verify data structure.

Use print(Model Training Started) and print(Model Training Completed) to track progress.

Use print(Model Saved Successfully) to confirm file storage.
