Model Training Execution Module (run_training.py)
Description
The run_training.py file is responsible for executing the complete model training process of the Crime Rate Prediction System. This module loads the historical crime dataset, preprocesses the data, trains the LSTM model, evaluates its performance, and saves the trained model for future crime prediction. It acts as the main training engine of the project and is executed whenever the model needs to be retrained with updated crime records.

Features
Dataset Loading
Loads crime datasets from CSV files.

Validates dataset structure and reads historical records.

Data Preprocessing
Handles missing values and removes duplicate records.

Performs normalization and converts data into a time-series format.

Model Training
Trains the LSTM Neural Network.

Learns historical crime patterns to generate the forecasting model.

Model Evaluation
Calculates training accuracy and generates loss metrics.

Measures overall prediction performance.

Model Saving
Saves trained model files and weights, enabling future prediction without requiring retraining.

Workflow
The training process follows this sequence:

Start Training.

Load the crime dataset.

Perform data cleaning and preprocessing.

Split the data into training and testing sets.

Train the LSTM model.

Evaluate the model performance.

Generate accuracy and loss graphs.

Save the trained model.

Training completed.

Input and Output
Inputs: The crime_data_india.csv file, which contains state/UT information, years, total IPC crimes, and historical crime statistics.

Outputs: Trained LSTM models saved in the saved_models directory, accuracy and loss graphs, and evaluation results including accuracy, loss, and performance metrics.

Common Errors
Error 1: Dataset Not Found
The error message is FileNotFoundError. This is because the dataset file path is incorrect. Verify that crime_data_india.csv exists in your data folder.

Error 2: TensorFlow Not Installed
The error message is ModuleNotFoundError: No module named 'tensorflow'. The TensorFlow library is missing; install it using the command: pip install tensorflow.

Error 3: Insufficient Data
The error message is ValueError. This occurs when the dataset does not contain enough records for training. Use a larger historical crime dataset.

Error 4: Model Saving Failed
The error message is PermissionError. The application cannot write model files to the directory. Verify the save directory permissions.

Error 5: Memory Error
The error message is MemoryError. This happens if the large dataset exceeds system memory. Reduce the batch size or use a machine with more RAM.

Debugging Guide
To debug the training process, use these print statements:

Use print(Dataset Loaded Successfully) to verify data ingestion.

Use print(df.shape) to check dataset size.

Use print(df.isnull().sum()) to check for missing values.

Use print(X_train.shape) and print(y_train.shape) to verify the training data dimensions.

Use print(Training Started) and print(Training Completed) to monitor progress.

Use print(Model Saved Successfully) to confirm storage.

Advantages and Limitations
Advantages: Automated training, deep learning-based forecasting, accurate trend prediction, and model reusability.

Limitations: Requires large historical datasets, training can be time-consuming, accuracy is highly dependent on data quality, and the model requires periodic retraining.
